# AMECxSV

Anonymous code release for the calibration, fusion, abstention, control, and
side-validation experiments reported in the AMECxSV submission.

This repository contains source code only. It does not include paper sources,
figures, speech recordings, protocols derived from restricted data, speaker
embeddings, trial scores, pretrained checkpoints, model weights, or generated
result tables.

## Paper-to-code map

### Trial-level systems

| Paper system | Implementation |
|---|---|
| C0 raw score | `experiments/c0.py`, `experiments/c0_c1_c2.py` |
| C1 logistic calibration | `experiments/c1.py`, `experiments/c0_c1_c2.py` |
| C2 PAV/isotonic calibration | `experiments/c2.py`, `experiments/c0_c1_c2.py` |
| C3 score + QMF | `experiments/c3.py` |
| C4 score + language match | `experiments/c4.py` |
| C5 score + duration reliability | `experiments/c5.py` |
| C8 multi-score control | `experiments/c8.py` |
| C10 metadata-aware fusion | `experiments/c10.py` |
| C10 feature ablation | `experiments/c10_ablation.py` |

The shared rich-feature MLP is implemented in
`experiments/rich_mlp.py`. Metric computation, fixed-split loading, affine LLR
calibration, and bootstrap utilities are in `experiments/common.py`.
Confidence-based abstention is implemented in `experiments/selective.py`.

### Additional evidence

| Analysis | Implementation |
|---|---|
| C8/C10 risk--coverage curves | `experiments/c8_coverage_curve.py`, `experiments/c10_coverage_curve.py` |
| Shuffled-metadata control | `experiments/c10_metadata_controls.py` |
| Metadata-only negative control | `experiments/c10_metadata_controls.py` |
| Coverage-curve AURC | `experiments/c10_metadata_controls.py` |
| Same-/cross-language analysis | `experiments/c13.py` |
| Duration-stratified analysis | `experiments/c13.py` |
| Speaker-clustered paired bootstrap | `experiments/c13.py` |

### C9 side validation

The separate multi-enrollment/group-level C9 experiment is implemented in
`experiments/c9.py`. Its compact input table is generated by
`data_prep/build_c9_multi_enroll_dataset.py`. C9 is not mixed with the main
trial-level comparison.

### External systems

The matched external-system pipeline is under `external/`:

- official TidyVoice SimAM-ResNet34 score extraction;
- LI-MSV w2v-BERT score extraction;
- protocol alignment and raw-score evaluation;
- single-score and dual-score AMEC calibration;
- confidence-based rejection on the calibrated external scores.

Third-party repositories and checkpoints are intentionally excluded. Their
locations are configured in `external/config/baselines.yaml`.

### Similarity and architecture checks

The negative experiments discussed in the limitations are included under
`similarity/` and in `experiments/c12.py`. They cover cosine, centered cosine,
whitened cosine, WCCN cosine, LDA cosine, negative Mahalanobis, AS-Norm, and
the tested gated/residual fusion model.

### Data preparation and fixed encoders

`data_prep/` contains protocol construction, speaker-disjoint splitting,
trial scoring, wide-table construction, C9 construction, and the VoxCeleb1B
sanity-check pipeline.

`embedding_extraction/` contains wrappers for the six fixed encoders:

1. SpeechBrain ECAPA-TDNN;
2. WeSpeaker ResNet34;
3. FunASR CAM++;
4. FunASR ERes2NetV2;
5. WavLM Base;
6. WavLM Base+.

No pretrained encoder files are distributed.

## Installation

Python 3.10 or newer is required. The MLP experiments require a CUDA-enabled
PyTorch installation.

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Install the optional fixed-encoder dependencies when extracting embeddings:

```bash
pip install -r requirements-embeddings.txt
```

On Windows:

```powershell
.venv\Scripts\Activate.ps1
```

## Main trial table

The default input is:

```text
data/tidyvoice_trials.parquet
```

It can be overridden without changing code:

```bash
export AMECXSV_INPUT_TABLE=/path/to/tidyvoice_trials.parquet
```

Required columns:

```text
split
trial_id
label
target
qmf1
qmf2
qmf3
qmf4_sum
qmf4_diff
min_duration
duration_ratio
short_duration_risk
score_speechbrain_ecapa_tdnn_voxceleb
score_wespeaker_resnet34_cnceleb
score_funasr_campplus_cn_3k
score_funasr_eres2netv2_cn_200k
score_hf_wavlm_base_sv_voxceleb1
score_hf_wavlm_base_plus_sv_voxceleb1
```

`label` uses `1` for target trials and `0` for nontarget trials. `target` is
the binary language-match indicator. The `split` column contains calibration,
validation, and test partitions.

## Follow-up metadata table

The condition and clustered-bootstrap analyses additionally use:

```text
data/tidyvoice_trial_metadata.parquet
```

Override it with:

```bash
export AMECXSV_FOLLOWUP_TABLE=/path/to/tidyvoice_trial_metadata.parquet
```

Required columns:

```text
split
trial_id
enroll_speaker
test_speaker
enroll_language
test_language
language_pair
language_condition
enroll_duration_sec
test_duration_sec
min_duration
duration_ratio
```

The speaker and language identifiers are used only for grouped evaluation and
clustered bootstrap. They are never supplied to the calibration MLP.

## Running experiments

List available tasks:

```bash
python run.py --list
```

Run the main C0--C5, C8, C10, and C10-ablation experiments:

```bash
python run.py --main
```

Run the metadata controls, risk--coverage curves, condition analysis, and
speaker-clustered bootstrap:

```bash
python run.py --followup
```

Run C9 separately because it uses a different group-level table:

```bash
python run.py c9
```

Run the multi-similarity negative experiment separately:

```bash
python run.py similarity_fusion
```

Generated outputs are written under ignored output directories and must not be
committed to an anonymous review repository.

## Reproducibility constraints

- Calibration, validation, and test partitions are fixed before training.
- Model selection and affine LLR calibration use validation data only.
- Confidence thresholds are selected on validation confidence.
- The test partition is not used for fitting or threshold selection.
- Metadata permutation is performed independently within each split.
- Clustered bootstrap resamples enrollment-speaker clusters and applies the
  same sampled weights to C8 and C10.
- C8 and C10 use the same MLP architecture and optimization setting.

## Anonymous-release checklist

- No author names, affiliations, email addresses, local absolute paths, or Git
  history are included.
- No dataset rows, trial IDs, speaker IDs, embeddings, scores, checkpoints, or
  generated metrics are included.
- Data, outputs, checkpoints, third-party repositories, and local environment
  files are excluded by `.gitignore`.
